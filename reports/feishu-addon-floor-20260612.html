Kernel dependency floor for Feishu addon fix

Why

lingtai-feishu 0.2.0 on PyPI predates the WebSocket event-loop fix and still satisfies the old loose kernel dependency lingtai-feishu>=0.1.0. Fresh installs can therefore resolve to a stale Feishu addon even after the kernel is upgraded.

Change

lingtai-feishu>=0.1.0  →  lingtai-feishu>=0.2.1

This makes fresh dependency resolution reject the stale 0.2.0 package once 0.2.1 is published.

Scope

Dependency metadata only. No runtime code changes in this PR.

Validation

git diff --check

Passed locally.

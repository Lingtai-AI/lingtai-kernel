Before and after

Before: manager-owned lifetime

  1. Agent manager starts a child and keeps transient process handles.
  2. Refresh/relaunch destroys the manager's in-memory ownership.
  3. A later poll can see files or a PID, but cannot recover exact wait status.
  4. Cancellation and reminders can race terminal truth or report premature success.
→

After: supervisor-owned lifetime

  1. Manager atomically creates durable job state with a finite start lease and bounded successful-return handoff.
  2. Supervisor claims and rechecks the start lease before Popen; parent reaping or later dead/expired proof closes every preclaim exit.
  3. Supervisor owns Popen, logs, whole-group TERM→KILL quiescence, exact wait(), and terminal commit.
  4. Any later manager rehydrates state; status: ok is permitted only while the return handoff still linearizes safely.

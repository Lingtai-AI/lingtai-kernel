# intrinsics/soul

> **Maintenance:** see the `lingtai-kernel-anatomy` skill. **Coding agents** update this file in the same commit as code changes. **LingTai agents** report drift as issues (mail or `discussions/<name>-patch.md`); do not silently fix.

Inner voice and soul-flow. Four agent-callable actions (`inquiry`,
`config`, `voice`, `flow`) â€” `flow` also fires on a wall-clock timer
mechanically. Voluntary `flow` calls return immediately with a success
acknowledgement; the actual voices arrive shortly after as a separate
involuntary `soul(action='flow')` pair (same shape as a timer-fired
one). A fire lock (`agent._soul_fire_lock`) gates concurrent fires so
the timer path and a voluntary call never run simultaneously.

## Components

- `soul/__init__.py` â€” public intrinsic surface. `get_description` (`soul/__init__.py:75-77`), `get_schema` (`soul/__init__.py:80-116`), `handle` (`soul/__init__.py:119-193`) (the dispatcher). The dispatcher accepts `action='flow'` from the agent: it try-acquires `agent._soul_fire_lock` non-blocking; on collision returns `{"error": "soul flow ongoing, request rejected"}`; otherwise releases the lock immediately and spawns a daemon thread running `_run_consultation_fire(agent)` (which re-acquires the lock under the same gate). The voluntary call's tool result is a small synthesized success acknowledgement, not the voices â€” the real voices land later as a separate involuntary `soul(action='flow')` pair via `tc_inbox`. Re-exports constants from `config` (`soul/__init__.py:24-31`), private helpers from `config` (`soul/__init__.py:33-41`), consultation pipeline from `consultation` (`soul/__init__.py:43-60`), `soul_inquiry` and `_run_inquiry` from `inquiry` (`soul/__init__.py:62-63`), and flow functions from `flow` (`soul/__init__.py:65-72`).
- `soul/config.py` â€” config and voice handling. `_handle_config` (`soul/config.py:27-113`) and `_handle_voice` (`soul/config.py:115-220`) dispatch `action='config'` and `action='voice'`. `_build_soul_system_prompt` (`soul/config.py:339-361`) resolves voice profiles to system prompts. `_persist_soul_config` (`soul/config.py:223-262`) and `_persist_soul_voice` (`soul/config.py:265-307`) write to `manifest.soul.*` in `init.json`. `_atomic_write_init` (`soul/config.py:310-336`) is the shared atomic JSON write helper. Constants: `SOUL_DELAY_MIN_SECONDS = 30.0` (`soul/config.py:11`), `CONSULTATION_PAST_COUNT_MIN = 0` (`soul/config.py:14`), `CONSULTATION_PAST_COUNT_MAX = 5` (`soul/config.py:15`), `SOUL_VOICE_BUILTINS` (`soul/config.py:20`), `SOUL_VOICE_PROMPT_MAX = 4000` (`soul/config.py:24`).
- `soul/consultation.py` â€” LLM call mechanics for past-self and current-self consultations. Constants: `_CONSULTATION_SYSTEM_PROMPT` (`soul/consultation.py:13-27`) â€” single-pass system prompt that frames the consultation as "advise your present self": tool schemas remain active, but every tool call is intercepted as a *recommendation channel* (the call name, args, and adjacent reasoning are forwarded to present-self via `_flatten_v3_for_pair`'s `Wanted to: tool(args)` lines). `_CONSULTATION_TOOL_REFUSAL` (`soul/consultation.py:33-39`) â€” sent as `ToolResultBlock.content` after every intercepted call. Confirms receipt of the recommendation (so the model doesn't think the call failed and retry the same call) and re-grounds with the full system prompt. `_CONSULTATION_MAX_ROUNDS = 3` (`soul/consultation.py:41`); `_DIARY_CUE_TOKEN_CAP = 10_000` (`soul/consultation.py:42`); `_CUE_EVENT_TYPES = ("diary", "thinking")` (`soul/consultation.py:75`) â€” both event types are mixed into the cue; `_CUE_EVENT_TYPES_BYTES` (`soul/consultation.py:80`) â€” UTF-8 byte tokens for the substring fast-path; `_REVERSE_READ_CHUNK = 64 * 1024` (`soul/consultation.py:85`). `_send_with_timeout` (`soul/consultation.py:45-72`) wraps LLM calls with a daemon thread. `_iter_lines_reverse` (`soul/consultation.py:88-129`) â€” tail-first byte-chunked line iterator over a JSONL file; relies on the append-only one-record-per-line invariant and on `\n`'s UTF-8 safety. `_render_current_diary` (`soul/consultation.py:132-200`) builds the time-anchored cue from recent **diary AND thinking** entries; uses `_iter_lines_reverse` with the substring fast-path (`_CUE_EVENT_TYPES_BYTES`) to skip JSON parsing on non-cue lines, stops once `_DIARY_CUE_TOKEN_CAP` is reached. Cost is O(recent cue entries) not O(file size). `_write_soul_tokens` (`soul/consultation.py:203-223`) appends token-ledger entries. `_load_snapshot_interface` (`soul/consultation.py:226-255`) loads a `ChatInterface` from a snapshot file. `_fit_interface_to_window` (`soul/consultation.py:258-344`) tail-trims a `ChatInterface` to a token budget. `_kind_for_source` (`soul/consultation.py:347-351`) maps source labels to prompt kinds (`"insights"` vs `"past"`); kept for tests/future use. `_build_consultation_cue` (`soul/consultation.py:354-377`) builds a localized cue from the `soul.consultation_cue_*` i18n templates; **not currently called from `_run_consultation`** â€” kept for tests/future re-wiring. The current design sends the raw `_render_current_diary` output as the spark for both insights and past branches; the cue templates are dormant. `_run_consultation` (`soul/consultation.py:380-481`) runs one substrate+spark consultation with refusal loop; bails when the diary is empty (no spark = no consultation), then sends the raw diary verbatim as the next-input. `_list_snapshot_paths` (`soul/consultation.py:484-492`) lists snapshot files. `_run_consultation_batch` (`soul/consultation.py:495-566`) orchestrates parallel 1+K consultations (1 insights + K past-self). `build_consultation_pair` (`soul/consultation.py:569-604`) builds the synthetic `(ToolCallBlock, ToolResultBlock)` pair sent back to present-self.
- `soul/inquiry.py` â€” on-demand inquiry. `soul_inquiry` (`soul/inquiry.py:9-61`) clones conversation (text+thinking only), sends question, returns answer. `_run_inquiry` (`soul/inquiry.py:64-77`) runs `soul_inquiry` and persists the result via `flow._persist_soul_entry`.
- `soul/flow.py` â€” soul-flow cadence and fire orchestration. This is the trunk of the soul package â€” both `inquiry.py` and the kernel hooks depend on it. `_start_soul_timer` (`soul/flow.py:19-35`), `_cancel_soul_timer` (`soul/flow.py:38-42`), `_soul_whisper` (`soul/flow.py:45-65`), `_persist_soul_entry` (`soul/flow.py:68-85`), `_append_soul_flow_record` (`soul/flow.py:87-93`), `_flatten_v3_for_pair` (`soul/flow.py:95-123`) â€” flattens consultation voice blocks into a `{source, voice, thinking}` payload; `ToolCallBlock`s become `Wanted to: <tool>(<args>)` lines appended to voice text (this is the channel through which past/insights tool-call recommendations reach present-self). `_soul_fire_allowed` (`soul/flow.py:126-130`) â€” true only when state âˆˆ {ACTIVE, IDLE}. `_run_consultation_fire` (`soul/flow.py:133-280`) â€” runs one consultation batch and persists the result; gates on `agent._soul_fire_lock` (try-acquire non-blocking, releases in `finally`) so timer-fired and voluntary calls can never run simultaneously. If the lock is held the call silently logs `consultation_skipped_inflight` and returns. `_rehydrate_appendix_tracking` (`soul/flow.py:283-318`).

## Connections

- `flow.py` is the trunk: both `inquiry.py` (imports `_persist_soul_entry` from `.flow`) and the kernel hooks (`_start_soul_timer`, `_cancel_soul_timer`, `_run_consultation_fire`, `_rehydrate_appendix_tracking`) depend on it.
- `__init__.py` imports from `config`, `inquiry`, `consultation`, and `flow` for dispatch and re-exports.
- `inquiry.py` imports `_build_soul_system_prompt` from `config`, `_send_with_timeout` + `_write_soul_tokens` from `consultation`, and `_persist_soul_entry` from `flow`.
- `flow.py` imports `_render_current_diary`, `_run_consultation_batch`, `build_consultation_pair` from `consultation` (sibling, within `flow._run_consultation_fire`).
- `config.py` and `consultation.py` are leaves â€” no intra-package imports.
- All modules use `i18n.t()` for localized strings.
- `consultation.py` reads snapshots written by `psyche._write_molt_snapshot` via `_load_snapshot_interface` and uses `llm.interface` block types throughout.

## State

- `config.py` mutates `init.json` (manifest.soul.*) for cadence and voice config via `_atomic_write_init`.
- `consultation.py` reads `history/snapshots/snapshot_*.json` and tail-reads `logs/events.jsonl` (filtered to `type âˆˆ {"diary","thinking"}`) for the cue, writes token-ledger entries via `_write_soul_tokens`.
- `flow.py` writes `logs/soul_flow.jsonl` (`soul/flow.py:87-93`) and `logs/soul_inquiry.jsonl` (via `_persist_soul_entry`), enqueues on `tc_inbox` (via `_run_consultation_fire`). Soul is the **canonical advanced producer** for involuntary tool-call pairs â€” it builds `InvoluntaryToolCall` directly with `coalesce=True` (latest voice wins when the timer fires repeatedly during a busy stretch) and `replace_in_history=True` (single-slot wire history â€” the prior pair is removed before the new one is spliced). Simple notification producers (mail, daemon, MCP) use the higher-level `agent._enqueue_system_notification(...)` helper instead. See root `ANATOMY.md` "Involuntary tool-call pairs".
- `agent._soul_fire_lock` (`threading.Lock` initialized in `base_agent/__init__.py`) is the in-flight gate for soul-flow consultation fires. Held for the lifetime of one `_run_consultation_fire` invocation; tried non-blocking by both the timer path (`_soul_whisper`) and the voluntary path (`handle(action='flow')`). On contention the timer path silently skips (logs `consultation_skipped_inflight`); the voluntary path returns `{"error": "soul flow ongoing, request rejected"}` so the agent gets a clear signal rather than a silent no-op.
